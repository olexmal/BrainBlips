# Project Instructions

## Code Style

- Use TypeScript for all new files
- Prefer functional components in React
- Use snake_case for database columns
- Follow Docusaurus best practices for documentation
- Use proper frontmatter with title, sidebar_position, description, tags, and keywords

## Architecture

- Follow the repository pattern for documentation organization
- Keep business logic in service layers
- Use MCP (Model Context Protocol) for AI agent integrations
- Maintain clear separation between development and production agents

## Documentation Standards

- Use Mermaid diagrams for system architecture visualization
- Include proper admonitions (:::tip, :::warning, :::note) for enhanced readability
- Follow consistent markdown formatting with tables and code blocks
- Maintain comprehensive cross-references between related documents

## AI Agent Guidelines

- Configure MCP servers with proper JSON structure
- Document all agent capabilities and functions clearly
- Implement safety features for production network operations
- Use sequential thinking patterns for complex problem-solving

## Project Structure

- Organize documentation in logical categories (AI, Docker, Optical Network Copilot, Meta)
- Use sidebar navigation for easy content discovery
- Maintain version control with proper Git workflows
- Keep configuration files in project root
